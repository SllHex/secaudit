# Code of Conduct

## Our Pledge

We want SecAudit to be a welcoming, professional, and respectful open-source project for maintainers, contributors, and users.

Participants are expected to:

- be respectful and constructive
- assume good intent
- focus feedback on code, design, docs, and behavior
- prioritize user safety and responsible security tooling

## Expected Behavior

- Use welcoming and inclusive language.
- Offer actionable technical feedback.
- Respect different levels of experience.
- Keep security discussions defensive, legal, and non-intrusive.

## Unacceptable Behavior

- Harassment, discrimination, or personal attacks
- Doxxing or sharing private information
- Trolling, intimidation, or repeated hostile conduct
- Requests to add offensive, exploitative, brute-force, or abusive behavior to the tool

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or contributions that violate this Code of Conduct.

## Reporting

If you need to report unacceptable behavior, please contact the maintainers through the repository security or contact channels listed in `SECURITY.md`.
